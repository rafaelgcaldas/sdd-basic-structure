# Como Executar

Este arquivo contém as instruções padrão que valem para qualquer especificação do projeto.

## Regras de execução

1. Leia `Objetivo` e `Contexto Técnico` antes de iniciar.
2. Execute as tasks na ordem em que aparecem dentro de cada grupo.
3. Ao concluir uma task, marque o checkbox e adicione a evidência logo abaixo.
4. Nunca remova uma task.
5. Se uma task não puder ser executada, registre o motivo na evidência.
6. Só considere a spec concluída quando todos os checkboxes estiverem marcados.
7. Quando uma task indicar uma ferramenta ou automação preferida (skill, script, gerador), use-a como implementação principal.
8. Se a ferramenta indicada não cobrir o caso inteiro, aplique-a até onde fizer sentido e registre o desvio na evidência.

## Formato da evidência

Use sempre este padrão:

```md
- [x] descrição da task
  > ✅ YYYY-MM-DD HH:MM — o que foi feito, decisões tomadas, desvios do plano
```

## Intenção

Estas regras existem para manter a execução rastreável sem transformar a spec em documentação pesada.
